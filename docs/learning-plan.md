# Godot 4 학습 플래닝 (2D 횡스크롤 플랫포머 목표)

완전 입문자 → 자작 2D 플랫포머 완성까지의 단계별 계획.
각 항목은 진행하면서 체크. "모든 내용을 다 알 필요는 없다"는 전제로, 막히면 일단 넘어가고 나중에 공식 문서로 보강.

---

## Phase 0 — 환경 준비

목표: 도구를 손에 익히고 첫 화면을 띄운다.

- [ ] [Godot 공식 사이트](https://godotengine.org/download)에서 macOS(Apple Silicon)용 Godot 4.x 다운로드
- [ ] 에디터 첫 실행, 새 프로젝트 만들기/열기
- [ ] 인터페이스 둘러보기: 씬(Scene) 독, 인스펙터(Inspector), 파일시스템(FileSystem), 출력(Output)
- [ ] 2D / 3D / 스크립트 작업 화면 전환 익히기

> 참고: 입문서 1장과 공식 문서 "에디터 첫 인상"을 함께 보면 빠르다.

## Phase 1 — 입문서: 『고도 엔진 4: 게임 개발 프로젝트』

목표: 게임을 따라 만들며 Godot의 작업 흐름과 감각을 익힌다. 이해 80%면 충분, 일단 끝까지 만든다.

- [ ] **1장. Godot 4 소개** — 엔진 철학, 노드/씬 개념, 에디터 기본
- [ ] **2장. Coin Dash (첫 2D 게임)** — 노드·씬 구성, 스프라이트, 입력, 충돌, 점수 UI
- [ ] **3장. Space Rocks (2D 아케이드)** — `RigidBody2D` 물리, 커스텀 입력, 파티클, 화면 래핑
- [ ] ⭐ **4장. Jungle Jump (2D 플랫포머)** — 목표와 직결. `CharacterBody2D`, 중력/점프, 타일맵 레벨, 적, 이동 발판
- [ ] **5장 이후 (3D / 무한 러너 / 미니골프 등)** — 가볍게 훑기 (선택, 2D 집중이면 건너뛰어도 됨)

> 4장(Jungle Jump)이 최종 목표 장르와 가장 가깝다. 여기서 만든 결과물을 Phase 4 자작 프로젝트의 출발점으로 삼는다.

## Phase 3 — 플랫포머 심화

목표: 자작 게임에 필요한 기술을 토픽별 미니 예제로 확보한다. 공식 문서 "튜토리얼" 섹션에서 골라 학습.

- [ ] **이동/물리**: `CharacterBody2D`, `move_and_slide()`, 중력·점프·코요테 타임
- [ ] **레벨**: TileMap / TileSet, 충돌 레이어와 마스크
- [ ] **카메라**: `Camera2D`, 패럴랙스 배경(ParallaxBackground)
- [ ] **애니메이션**: `AnimatedSprite2D`, `AnimationPlayer`, 간단한 상태 머신(idle/run/jump/fall)
- [ ] **적/상호작용**: 적 AI, 체력·대미지, 수집 아이템, `Area2D`
- [ ] **게임 구조**: 씬 전환, 일시정지, 저장/불러오기, 사운드(`AudioStreamPlayer`)

각 토픽마다 `experiments/` 아래에 작은 테스트 씬을 만들어 직접 확인.

---

## Phase 4 — 목표 프로젝트: 자작 2D 횡스크롤 플랫포머

목표: 플레이 가능한 게임 한 편을 완성하고 빌드한다.

- [ ] **기획**: 한 줄 컨셉, 핵심 재미, 조작, 1~2개 스테이지 분량으로 축소
- [ ] **프로토타입**: 이동·점프·충돌만 되는 그레이박스 레벨
- [ ] **콘텐츠**: 타일셋 아트, 적, 아이템, 장애물, 골인 지점
- [ ] **폴리시**: 애니메이션, 사운드, 파티클, UI(체력/점수/타이머)
- [ ] **마무리**: 타이틀/게임오버 화면, 씬 전환
- [ ] **빌드/배포**: export preset 설정, macOS(또는 웹) 빌드 만들기
