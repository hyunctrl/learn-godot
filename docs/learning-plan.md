# Godot 4 학습 플래닝 (2D 횡스크롤 플랫포머 목표)

> 🎯 **북극성**: 대학 졸업 스타트업 수준으로 게임을 개발해 **직접 퍼블리싱하는 경험**을 갖는다. 완벽한 이해보다 *완성과 출시*가 우선.

완전 입문자 → 자작 2D 플랫포머 완성 → 출시까지의 단계별 계획.
각 항목은 진행하면서 체크. "모든 내용을 다 알 필요는 없다"는 전제로, 막히면 일단 넘어가고 나중에 공식 문서로 보강.
모르는 인접 분야(기획·디자인·레벨·QA)는 [roadmap.md](roadmap.md)의 가지치기 트랙에서 보강, 자료는 [resources.md](resources.md).

## Phase 1 — 공식 튜토리얼: 첫 2D 게임 (Dodge the Creeps)

목표: 노드·씬·**시그널·인스턴스**·UI·간단 게임플레이를 한 바퀴 돈다. 책 없이 [공식 문서](https://docs.godotengine.org/ko/4.x/getting_started/first_2d_game/index.html)로 진행하며 튜터가 전 과정을 가이드한다.

- [ ] **프로젝트 설정** — 새 프로젝트, 창 크기 480×720, 스트레치 모드 `canvas_items`/비율 `keep`, 공식 아트 에셋 가져오기
- [ ] **플레이어 씬** — `Area2D` + `AnimatedSprite2D` + 충돌, 이동 스크립트(입력)
- [ ] **적(Mob) 씬** — `RigidBody2D` 적, 랜덤 생성 준비
- [ ] **메인 씬 조립** — 플레이어·타이머·스폰 경로 배치, 적 **인스턴스** 생성
- [ ] **HUD** — 점수·메시지 라벨, **시그널**로 게임 흐름 연결
- [ ] **마무리** — 사운드, 게임 오버/재시작, 실행 확인

> 필요한 외부 자료: **공식 무료 아트 에셋 팩** 하나뿐(튜터가 링크 제공). 코드·단계는 튜터가 전부 안내.
> 책 『고도 엔진 4: 게임 개발 프로젝트』는 **선택적 보조** — 없어도 진행에 지장 없음.

## Phase 3 — 플랫포머 심화

목표: 자작 게임에 필요한 기술을 토픽별 미니 예제로 확보한다. 공식 문서 "튜토리얼" 섹션에서 골라 학습.

- [ ] **이동/물리**: `CharacterBody2D`, `move_and_slide()`, 중력·점프·코요테 타임
- [ ] **레벨**: TileMap / TileSet, 충돌 레이어와 마스크
- [ ] **카메라**: `Camera2D`, 패럴랙스 배경(ParallaxBackground)
- [ ] **애니메이션**: `AnimatedSprite2D`, `AnimationPlayer`, 간단한 상태 머신(idle/run/jump/fall)
- [ ] **적/상호작용**: 적 AI, 체력·대미지, 수집 아이템, `Area2D`
- [ ] **게임 구조**: 씬 전환, 일시정지, 저장/불러오기, 사운드(`AudioStreamPlayer`)

각 토픽마다 `experiments/` 아래에 작은 테스트 씬을 만들어 직접 확인.

## Phase 4 — 목표 프로젝트: 자작 2D 횡스크롤 플랫포머

목표: 플레이 가능한 게임 한 편을 완성하고 빌드한다.

- [ ] **기획**: 한 줄 컨셉, 핵심 재미, 조작, 1~2개 스테이지 분량으로 축소
- [ ] **프로토타입**: 이동·점프·충돌만 되는 그레이박스 레벨
- [ ] **콘텐츠**: 타일셋 아트, 적, 아이템, 장애물, 골인 지점
- [ ] **폴리시**: 애니메이션, 사운드, 파티클, UI(체력/점수/타이머)
- [ ] **마무리**: 타이틀/게임오버 화면, 씬 전환
- [ ] **빌드/배포**: export preset 설정, macOS(또는 웹) 빌드 만들기

## Phase 5 — 퍼블리싱 ★ (최종 목표)

목표: 만든 게임을 실제로 세상에 내놓는다. "출시 경험"이 이 전체 학습의 종착점.

- [ ] **클린 빌드**: export로 배포용 빌드 생성, **내 개발 PC가 아닌 환경에서 실행 확인** (출시 작업의 90%)
- [ ] **itch.io 출시**: 무료 계정 → 프로젝트 페이지(제목·설명·커버 315:250·스크린샷 3~5장·GIF/트레일러) → zip 업로드 → 공개. 데모/무료로 먼저 올려 피드백 수집
- [ ] **플레이테스트 반영**: 실제 플레이어 피드백으로 버그·난이도 수정 (QA 트랙과 연계)
- [ ] **(선택) Steam 정식 출시**: Steam Direct 등록($100, 신원·세금·계좌) → 스토어 페이지 → 'Coming Soon'으로 위시리스트 2~6개월 확보 → 심사(1~5영업일) → Next Fest 데모 활용
- [ ] **출시 후 운영**: 런치 후 24시간 피드백/버그 집중 대응, devlog·SNS(X/Reddit/Discord) 기록

> 통념: itch.io에 먼저 → 반응 좋으면 Steam. 자료는 [resources.md 6번](resources.md) 참고.
