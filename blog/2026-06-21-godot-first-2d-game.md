# Godot로 첫 2D 게임 만들기 — Dodge the Creeps 회고

게임 개발은 처음이다. 목표는 **2D 횡스크롤 플랫포머를 직접 만들어 출시**해보는 것. 그 첫걸음으로 Godot 4 공식 튜토리얼인 **"Dodge the Creeps"**(쏟아지는 적을 피하는 2D 게임)를 처음부터 끝까지 따라 만들었다. 이 글은 그 과정에서 **새로 배운 개념과 막혔던 지점**을 정리한 기록이다.

## 왜 Godot인가

- **무료·오픈소스**, 로열티 없음. 인디로 출시하기 부담 없음.
- **가볍고 빠르다.** 설치 100MB 남짓, 에디터가 즉시 뜬다.
- **2D가 강하다.** 횡스크롤 플랫포머 목표에 잘 맞는다.
- 언어는 **GDScript** — 파이썬과 비슷해서 입문하기 쉽다. (Godot 전용 언어다. C#을 쓰려면 ".NET" 버전을 받아야 하지만, 입문은 GDScript가 정석.)

## Godot의 핵심 사고방식: 노드 → 씬 → 트리

가장 먼저 깨야 했던 개념.

- **노드(Node)** = 기능 한 조각(스프라이트, 충돌체, 사운드, 타이머…). 레고 블록 같은 것.
- 노드를 부모-자식으로 쌓으면 **트리**가 되고, 그 묶음 하나가 **씬(Scene)**.
- 씬은 다른 씬 안에 **부품처럼 끼워 넣을 수 있다.** 이게 처음엔 헷갈렸는데, 개발자 관점으로 보니 명확했다:

> **씬 파일 = 클래스(또는 프리팹). 씬을 다른 씬에 넣는 것 = 인스턴스 생성(`new Player()`).**

여전히 구조는 **트리**다. 그래프처럼 얽히는 게 아니라, 트리 안에 트리를 끼워 넣는 중첩 구조. Unity의 프리팹과 같은 개념이라고 보면 된다.

실제로 이 게임도 `Player`, `Mob`, `HUD`를 각각 독립된 씬으로 만들고, `Main` 씬에 인스턴스로 조립했다. 적(코인 같은 반복 요소)은 씬 하나만 만들면 코드에서 몇 개든 찍어낼 수 있다.

## 게임을 조립하며 배운 것들

### 1. 시그널(Signal) = 이벤트 버스
노드끼리 직접 의존하지 않고, **"이벤트를 쏘면 구독자가 받는"** 느슨한 연결. 메시지 패싱/이벤트 버스와 같은 직관이다.

- 플레이어가 적에 닿으면 `hit` 신호를 쏘고, 메인이 그걸 받아 게임을 끝낸다.
- 시작 버튼은 직접 게임을 시작하지 않고 `start_game` 신호만 보낸다. 메인이 그 신호를 받아 `new_game()`을 실행 → **역할 분리**.
- 타이머의 `timeout`, Area2D의 `body_entered`도 전부 시그널. 에디터에서 시그널을 함수에 **드래그하듯 연결**할 수 있다는 게 인상적이었다.

### 2. 물리 바디: RigidBody2D vs Area2D
- **Area2D** = "닿았는지 감지"만 함(밀어내지 않음). 플레이어가 이걸 써서 적과의 접촉을 감지.
- **RigidBody2D** = 진짜 물리 바디. 적이 이걸 써서 속도를 받아 화면을 가로지른다.
- **충돌 레이어(Layer)와 마스크(Mask)**: "나는 몇 번 층에 있다(Layer)" vs "나는 몇 번 층과 부딪힌다(Mask)". 적의 Mask를 비우니 **적끼리는 통과**하면서도 플레이어는 여전히 적을 감지했다. (Area2D는 상대의 Layer만 보고 감지하기 때문)

### 3. 그룹(Group) = 태그
적을 `mobs` 그룹에 등록해두니, 게임오버 시 `get_tree().call_group("mobs", "queue_free")` 한 줄로 **모든 적을 한 번에 제거**할 수 있었다. "모든 적", "모든 코인"처럼 묶어서 명령할 때 강력하다.

### 4. 타이머와 UI
- `Timer` 노드로 적 생성 주기, 점수 증가, 시작 지연을 처리.
- `CanvasLayer` 위에 Label/Button을 올려 점수·메시지·시작 버튼을 띄웠다. 카메라가 움직여도 UI는 화면에 고정된다.
- `await timer.timeout` 으로 "게임오버 → 잠시 후 타이틀 → 버튼 표시" 같은 **순서 연출**을 구현.

## 막혔던 지점 (다음 사람을 위해)

- **애니메이션 이름 바꾸는 곳을 못 찾음.** → 인스펙터가 아니라 **에디터 아래쪽 SpriteFrames 패널**에서 `default`를 더블클릭해 바꾼다. 인스펙터엔 "리소스 연결"만 있다.
- **충돌 레이어 끄는 게 안 보임.** → `CollisionShape2D`가 아니라 **`RigidBody2D` 루트 노드**를 선택해야 Layer/Mask가 나온다. 그리고 체크박스가 아니라 **작은 칸 격자**라 파란 칸을 클릭해 끈다.
- **적끼리 서로 통과를 못 함.** → RigidBody2D 기본 동작(물리 충돌). Mask를 비워 해결.
- **첫 실행이 느림.** → 정상. 첫 실행은 에셋 임포트·캐시 준비로 오래 걸리고, 두 번째부터 빨라진다.

## 결과

시작 버튼 → 플레이 → 점수 → 게임오버 → 재시작까지 도는 **완전한 2D 게임 한 편**을 완성했다. 코드는 길지 않았지만, 그 안에 **노드·씬·인스턴스·시그널·그룹·타이머·물리·UI** 라는 2D 게임의 핵심 뼈대가 전부 들어있었다.

## 다음 단계

목표인 **플랫포머**는 이 위에 "중력·점프(`CharacterBody2D`)"와 "타일맵 레벨"을 얹는 것이다. 절반은 이미 손에 넣은 느낌. 다음은 `CharacterBody2D`로 캐릭터를 중력 아래에서 움직이고 점프시키는 것부터 시작할 예정이다.
